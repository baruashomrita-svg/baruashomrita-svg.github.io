## How to publish on GitHub Pages (free, ~10 minutes)

Your GitHub username is **`baruashomrita-svg`**, so your site will live at **`https://baruashomrita-svg.github.io`** — the exact URL printed on your resume.

### Step 1 — Create the repository

1. Sign in at https://github.com/baruashomrita-svg
2. Click the green **"New"** button (top-left), or go directly to https://github.com/new
3. **Repository name** must be exactly: `baruashomrita-svg.github.io`
   - This exact naming convention is what tells GitHub to publish the repo as your personal site.
4. Set visibility to **Public**
5. Do **not** initialize with a README (you'll upload your own files instead)
6. Click **Create repository**

### Step 2 — Upload the portfolio files

1. On the new empty repo page, click **"uploading an existing file"** (the blue link in the middle of the page).
2. Open this `portfolio` folder on your computer.
3. **Important:** drag in the *contents* of the folder, not the folder itself. Select these items together:
   - `index.html`
   - `Shomrita_Barua_PublicHealth_Policy_Resume.pdf`
   - the `assets/` folder (with the three files inside)
4. Scroll down, leave the default commit message, and click **Commit changes**.

### Step 3 — Wait ~1–2 minutes, then visit your site

Open `https://baruashomrita-svg.github.io` in a new tab. The first deploy takes a minute or two; if you see a 404, refresh after another minute.

### Step 4 — Add the link everywhere

- **LinkedIn:** Edit profile → "Add profile section" → "Featured" → Add link → paste `https://baruashomrita-svg.github.io`. It shows up as a card on your profile.
- **Resume:** Already updated — the resume in this folder links to your portfolio URL.
- **Email signature:** Add `Portfolio: https://baruashomrita-svg.github.io` under your name.

## Alternative — Netlify Drop (instant, no Git knowledge needed)

If you want a faster preview before committing to GitHub:

1. Go to https://app.netlify.com/drop
2. Drag the entire `portfolio` folder onto the page.
3. You get a live URL immediately (something like `https://amazing-name-12345.netlify.app`).
4. You can claim it with a free Netlify account and rename it to something cleaner.

Note: a Netlify URL won't match the one printed on your resume unless you also update the resume — so GitHub Pages is recommended for consistency.

## Updating content later

Open `index.html` in any text editor (VS Code, Sublime, even Notepad). Each section is labeled with HTML comments like `<!-- ============== CAPSTONE ============== -->`. Edit the text, save the file, then re-upload it to your GitHub repo (drag-and-drop replaces the existing file).

To swap an infographic, replace the file in `assets/` with the same filename — no code changes needed.

## Optional — buy a custom domain later

If you want a more memorable URL like `shomritabarua.com`:

1. Buy the domain from Namecheap, Cloudflare Registrar, or Google Domains (~$10–15/year).
2. In your `baruashomrita-svg.github.io` repo, go to **Settings → Pages** and add the custom domain.
3. Follow GitHub's DNS instructions on the registrar side.
4. Update your resume to match.

This is purely optional — `https://baruashomrita-svg.github.io` is already perfectly professional.
